# Database Schema Documentation
## Smart Energy & Security Monitoring System

---

## MySQL Database — smart_energy_db

### Table: users
| Column     | Type         | Constraints                        |
|------------|--------------|------------------------------------|
| id         | INT          | PRIMARY KEY, AUTO INCREMENT        |
| name       | VARCHAR(255) | NOT NULL                           |
| email      | VARCHAR(255) | NOT NULL, UNIQUE                   |
| password   | VARCHAR(255) | NOT NULL (bcrypt hashed)           |
| role       | ENUM         | Admin / Warden / Viewer            |
| isActive   | BOOLEAN      | DEFAULT true                       |
| createdAt  | DATETIME     | AUTO                               |
| updatedAt  | DATETIME     | AUTO                               |

### Table: rooms
| Column      | Type         | Constraints                       |
|-------------|--------------|-----------------------------------|
| id          | INT          | PRIMARY KEY, AUTO INCREMENT       |
| room_number | VARCHAR(255) | NOT NULL, UNIQUE                  |
| floor       | INT          | NOT NULL                          |
| building    | VARCHAR(255) | DEFAULT 'Main Building'           |
| status      | ENUM         | active / inactive / maintenance   |
| occupancy   | ENUM         | occupied / vacant                 |
| createdAt   | DATETIME     | AUTO                              |
| updatedAt   | DATETIME     | AUTO                              |

### Table: devices
| Column      | Type         | Constraints                            |
|-------------|--------------|----------------------------------------|
| id          | INT          | PRIMARY KEY, AUTO INCREMENT            |
| device_id   | VARCHAR(255) | NOT NULL, UNIQUE (e.g. ROOM-101-ENERGY)|
| device_name | VARCHAR(255) | NOT NULL                               |
| type        | ENUM         | energy / motion / door / temperature   |
| room_id     | INT          | FOREIGN KEY → rooms.id                 |
| status      | ENUM         | active / inactive / offline            |
| last_seen   | DATETIME     | Updated on each sensor reading         |
| createdAt   | DATETIME     | AUTO                                   |
| updatedAt   | DATETIME     | AUTO                                   |

### Relationships
- rooms → devices : One room has many devices (1:N)
- users : Standalone table, role-based access only

## MongoDB Database — smart_energy_db

### Collection: sensordatas
| Field      | Type    | Description                              |
|------------|---------|------------------------------------------|
| _id        | ObjectId| Auto generated by MongoDB                |
| device_id  | String  | Must match device_id in MySQL devices    |
| room_id    | Number  | Must match id in MySQL rooms             |
| type       | String  | energy / motion / door / temperature     |
| value      | Mixed   | Number / Boolean / String based on type  |
| unit       | String  | kWh / celsius / null                     |
| timestamp  | Date    | DEFAULT Date.now                         |

### Why Two Databases?
- **MySQL** stores structured data: users, rooms, devices (rarely changes)
- **MongoDB** stores high-frequency sensor readings (thousands per day)
- MongoDB handles unstructured/flexible sensor values better than SQL
- This is the standard IoT architecture pattern

## API Endpoints Summary

| Method | Endpoint                        | Auth    | Description              |
|--------|---------------------------------|---------|--------------------------|
| POST   | /api/auth/register              | None    | Register new user        |
| POST   | /api/auth/login                 | None    | Login, get JWT token     |
| GET    | /api/users/me                   | Any     | Get own profile          |
| GET    | /api/users                      | Admin/Warden | List all users      |
| GET    | /api/rooms                      | Any     | List all rooms           |
| POST   | /api/rooms                      | Admin/Warden | Create room         |
| GET    | /api/devices                    | Any     | List all devices         |
| POST   | /api/devices                    | Admin/Warden | Register device     |
| POST   | /api/sensors/data               | None    | Save sensor reading      |
| GET    | /api/sensors/latest/:roomId     | Any     | Latest readings per room |
| GET    | /api/sensors/history            | Any     | Sensor history + filters |
